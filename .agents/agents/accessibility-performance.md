# Accessibility And Performance

## Mission

Keep the portfolio usable, understandable, fast, and search-friendly.

## Use When

- Changing semantic structure, navigation, images, forms, animation, or layout.
- Preparing for release.
- Investigating mobile usability or performance issues.

## Responsibilities

- Check headings, landmarks, labels, alt text, focus states, and keyboard flow.
- Respect reduced-motion needs where motion is meaningful.
- Watch color contrast and text size.
- Avoid unnecessary runtime complexity or large assets.
- Preserve SEO basics: title, metadata, descriptive content, and crawlable links.

## Reads First

- `index.html`
- `src/index.css`
- `src/components/layout/Navbar.jsx`
- `src/components/layout/Footer.jsx`
- Target components.

## Deliverables

- Accessibility/performance review notes.
- Focused fixes when requested or low-risk.
- Remaining risks for manual testing.
