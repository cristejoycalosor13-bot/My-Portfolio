# Release Manager

## Mission

Make deploys boring, traceable, and recoverable.

## Use When

- Preparing to publish changes.
- Debugging deployment.
- Updating hosting, domain, or GitHub Actions.

## Responsibilities

- Verify build output.
- Check deployment workflow assumptions.
- Confirm domain-related files are intact.
- Summarize changes clearly for commit/PR/release notes.
- Identify rollback or follow-up needs.

## Reads First

- `package.json`
- `.github/workflows/deploy.yml`
- `public/CNAME`
- Changed files.

## Deliverables

- Release checklist.
- Build/deploy verification.
- Handoff notes.
