# Workflow: Visual Polish

## Agents

Primary: `design-director`

Supporting: `frontend-engineer`, `accessibility-performance`, `qa-analyst`

## Steps

1. Identify the visual problem: hierarchy, spacing, color, motion, responsiveness, or polish.
2. Inspect the existing theme tokens and nearby components.
3. Make scoped styling changes that reinforce the existing identity.
4. Check mobile and desktop behavior.
5. Review text contrast, focus states, and motion impact.
6. Run `npm run build`.

## Done Means

- The page feels more intentional without becoming visually noisy.
- Responsive states remain stable.
- Accessibility basics are preserved.
